# Installation

Geval is a **single binary** — no npm, no pip, no runtime dependencies. Install it once and run it anywhere (local or CI).

---

## 1. Download a release (recommended)

**Releases:** [github.com/geval-labs/geval/releases](https://github.com/geval-labs/geval/releases)

Pick the binary for your OS and architecture:

| OS      | Architecture | Download |
|---------|--------------|----------|
| Linux   | x86_64       | [geval-linux-x86_64](https://github.com/geval-labs/geval/releases/latest/download/geval-linux-x86_64) |
| macOS   | Apple Silicon (M1/M2/M3) | [geval-macos-aarch64](https://github.com/geval-labs/geval/releases/latest/download/geval-macos-aarch64) |
| macOS   | Intel        | [geval-macos-x86_64](https://github.com/geval-labs/geval/releases/latest/download/geval-macos-x86_64) |
| Windows | x86_64       | [geval-windows-x86_64.exe](https://github.com/geval-labs/geval/releases/latest/download/geval-windows-x86_64.exe) |

**One-liner (Linux / macOS):**

```bash
# Linux
curl -sSL https://github.com/geval-labs/geval/releases/latest/download/geval-linux-x86_64 -o geval
chmod +x geval

# macOS (Apple Silicon)
curl -sSL https://github.com/geval-labs/geval/releases/latest/download/geval-macos-aarch64 -o geval
chmod +x geval

# macOS (Intel)
curl -sSL https://github.com/geval-labs/geval/releases/latest/download/geval-macos-x86_64 -o geval
chmod +x geval
```

**Add to PATH (optional but recommended):**

Move the binary to a directory on your PATH so you can run `geval` from anywhere:

```bash
# Example: user bin
mv geval ~/bin/geval

# Example: system-wide (may need sudo)
sudo mv geval /usr/local/bin/geval
```

**Verify:**

```bash
geval --help
```

## 2. Build from source

Use this if there’s no release for your platform or you want to build from the latest code.

**Prerequisites:** [Rust](https://rustup.rs/) (`curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`).

**From the repo root:**

```bash
git clone https://github.com/geval-labs/geval.git
cd geval
cargo build --release --manifest-path geval/Cargo.toml
```

Binary path: `geval/target/release/geval`. Copy to PATH if you like:

```bash
cp geval/target/release/geval /usr/local/bin/geval
```

---

## 3. Use in CI

**With a release binary (no Rust needed in CI):**

- Download the right artifact in your workflow (e.g. `geval-linux-x86_64` for `ubuntu-latest`).
- Run `./geval check --signals signals.json --policy policy.yaml --env prod`.

**With build from source:**

- Check out the repo, run `cargo build --release --manifest-path geval/Cargo.toml`, then run `geval/target/release/geval check ...`.

See [GitHub Actions](github-actions.md) for full workflow examples.

## FAQ

| Question | Answer |
|----------|--------|
| Is there an npm package? | No. Geval is a standalone binary. |
| Is there a pip package? | No. Geval is a standalone binary. |
| Do I need Node or Python to run Geval? | No. You only need the binary. (Your *signals* can still be generated by Node/Python scripts.) |
| When do I need Rust? | Only when building from source. |
